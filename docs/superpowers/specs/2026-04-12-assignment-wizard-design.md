# 대학배정 위자드 설계 스펙

## 개요

관리자가 대학에 운영자를 배정하는 워크플로우 도구. 기존 탭+테이블 조회 화면을 **스텝 위자드 + 대시보드** 구조로 전면 교체한다.

**핵심 흐름:** 대시보드(진행률 확인) → 배정 유형 선택 → 위자드 진입(용량설정 → 시뮬레이션 → 조정 → 적용) → 대시보드 복귀

## 배정 유형

| 유형 | 대상 데이터 | 대학 수 (예시) |
|------|-----------|--------------|
| 수시 | assignments 전체 | 498 |
| 정시 | assignments 전체 | 498 |
| 재외국민 | assignments 중 4년제 | 172 |
| 편입 | assignments 중 4년제 | 172 |
| 외국인 | assignments 중 4년제 | 172 |
| 대학원 | gradAssignments | 68 |
| PIMS | pimsAssignments | 89 |
| 성적산출 | scoreAssignments | 68 |
| 상담앱 | appAssignments | 40 |

## 화면 구성

### 1. 대시보드 (메인 페이지)

대학배정 메뉴 진입 시 보이는 화면. 서버 컴포넌트.

**상단: 유형별 진행률 카드 그리드**
- 9개 배정 유형 각각 카드로 표시
- 카드 내용: 유형명, 배정완료 수/전체 수, 진행률 바, 미배정 수
- 미배정이 있는 카드: 진행률 바 색상 warning/error
- 카드 클릭 → 해당 유형의 위자드 Step 1로 진입
- 100% 완료된 카드도 클릭 가능 (재조정용)

**하단: 운영자별 전체 워크로드 테이블**
- 행: 운영자 15명 (1그룹→6그룹 순서, 그룹 구분선)
- 열: 수시, 정시, 재외, 편입, 외국인, 대학원, PIMS, 성적, 상담앱, 합계
- 합계 열에 ProgressBar (평균 대비 과부하 색상 구분)
- 그룹별 소계 행

### 2. 위자드

클라이언트 컴포넌트. URL `?type=susi&step=1` 형태로 상태 관리.

#### Step 1: 배정 유형 + 용량 설정

**좌측 패널: 용량 설정**
- 배정 유형 선택 드롭다운 (이미 URL로 결정된 상태)
- 운영자별 최대 배정 수 입력 (number input)
- 그룹별로 묶어서 표시 (1그룹→6그룹)
- 기초자료 기반 **자동 추천값** 프리필 (관리자가 수정 가능)
- "기본값 초기화" 버튼

**우측 패널: 작년 실적 참고**
- 기초자료(연간 서비스 건수)에서 추출한 운영자별 실적
- 서비스ID 기준 대학별 건수
- 읽기 전용 참고 정보

**하단: [다음 →] 버튼**

#### Step 2: 시뮬레이션 실행 + 결과 미리보기

**시뮬레이션 실행 영역**
- [시뮬레이션 실행] 버튼 → 서버 액션 `runAutoAssign` 호출
- 실행 중 로딩 상태 표시

**결과 요약 배너**
- 대상 대학 수 → 배정 완료 → 신규 배정 → 배정 불가
- 각 숫자에 색상 코딩 (완료: green, 신규: yellow, 불가: red)

**운영자별 결과 바 차트**
- 1그룹→6그룹 순서
- 각 운영자: 이름 + ProgressBar + 현재/최대 표시
- 초과 시 빨간색

**하단: [← 이전] [결과 확인 & 조정 →]**

#### Step 3: 결과 확인 & 조정

**상단: 검색/필터 바**
- 대학명 검색 input
- 대분류 select (4년제, 전문대, 폴리텍 등)
- 지역 select

**메인 영역: 2패널 레이아웃 (8:4)**

왼쪽 (8/12): 대학 목록 테이블
- 열: 분류, 지역, 대학명, 엑셀 기준(원본 운영자), 배정 운영자, 변경 버튼
- 변경된 행 하이라이트 (tertiary 배경)
- 미배정 행 "미배정" 표시 (error 색상)
- [변경] 클릭 → 드롭다운으로 운영자 선택

우측 (4/12): 운영자 현황 패널
- 그룹별 운영자 + ProgressBar + 현재/최대
- 초과 운영자 빨간색 표시

**하단: 조건 변경 영역**
- 특정 운영자 max값 변경 input
- [조건 변경 후 재실행] 버튼 → 변경된 조건으로 시뮬레이션 재실행
- 기존 수동 조정은 유지, 미배정분만 재배정

**하단: [← 이전] [최종 확인 →]**

#### Step 4: 최종 확인 & 적용

**변경 요약**
- 신규 배정 건수
- 수동 조정 건수
- 최종 배정률 (N/M, %)

**운영자별 최종 배정 수 테이블**
- 변경 전 → 변경 후 비교 표시

**액션 버튼**
- [← 수정하기] — Step 3으로 돌아감
- [배정 적용] — `applyAssignments` 호출 → 대시보드로 복귀

## 데이터 흐름

### 기존 활용 (수정 금지)
- `fetchAssignments()` — SharePoint에서 엑셀 데이터 가져오기
- `runAutoAssign()` — 서버사이드 자동배정 알고리즘
- `applyAssignments()` — 배정 결과 DB 저장
- `manualAssign()` — 개별 수동 배정
- `getAssignmentResults()` — 기존 배정 결과 조회
- `getCapacitySettings()` — 용량 설정 조회
- `constants.ts` — 운영자 그룹 정의

### 신규 추가
- `sharepoint.ts`에 기초자료 시트 파싱 함수 추가
  - `fetchBaseData()`: "기초자료" 시트에서 운영자별 연간 서비스 건수 추출
  - 반환: `{ operatorStats: { name: string; totalServices: number; byType: Record<string, number> }[] }`

## 구현 우선순위

### 1차 (핵심)
- 대시보드 (진행률 카드 + 워크로드 테이블)
- 위자드 4단계 (용량설정 → 시뮬레이션 → 결과조정 → 적용)
- 개별 변경 + 조건 재실행
- 기초자료 연동 (자동 추천 + 참고 표시)

### 2차 (후속)
- 드래그앤드롭 배정 (`@dnd-kit`)
- 시뮬레이션 히스토리 저장/비교
- 배정 로그 타임라인 뷰

## 제약 사항

- `actions.ts`의 서버 액션 인터페이스는 변경하지 않음
- `constants.ts`의 운영자 그룹은 현재 상태 유지
- SharePoint API 호출 구조(`sharepoint.ts`의 `fetchAssignments`)는 유지, 기초자료 파싱만 추가
- 각 파일 400줄 이하
- TypeScript strict mode 준수
