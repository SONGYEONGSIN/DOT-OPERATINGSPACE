# 담당자 배정 (운영) 리디자인 스펙

## 목표

운영 > 담당자배정 페이지를 **분할 패널(Master-Detail)** 구조로 재설계하여, 운영자 기준 조회 / 대학 기준 조회 / 전체 현황 파악을 하나의 화면에서 양방향 탐색으로 처리한다.

## 현재 문제

- 검색하지 않으면 테이블 + 대학 인덱스만 보임 (조회 불편)
- 검색 결과가 전체 페이지를 대체하여 컨텍스트를 잃음
- 운영자→대학, 대학→운영자 간 탐색이 단절됨

## 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│  PageHeader + KPI 요약 (기존 유지)                        │
├──────────────┬──────────────────────────────────────────┤
│  왼쪽 패널    │  오른쪽 상세 패널                           │
│  (300px 고정) │  (flex-1)                                │
│              │                                          │
│ [운영자|대학]  │  운영자 선택 시:                           │
│  토글 버튼    │  - 프로필 헤더 (이름, 그룹, 총 건수)        │
│              │  - 유형별 섹션 카드                         │
│ 🔍 검색      │    (수시/정시/PIMS/대학원/성적/상담앱)       │
│              │  - 각 섹션 내 대학명 칩                     │
│ ┌──────────┐ │                                          │
│ │ 한효진 26│ │  대학 선택 시:                              │
│ │ 윤지혜 20│ │  - 대학 헤더 (이름, 대분류, 지역)            │
│ │ 박시현 18│ │  - 배정 정보 원카드                         │
│ │ ...      │ │    (수시/정시 운영자·개발자, 대학원,         │
│ └──────────┘ │     PIMS, 성적, 상담앱)                    │
│              │  - 운영자명 클릭 → 운영자 뷰 전환            │
├──────────────┴──────────────────────────────────────────┤
│  (KPI 카드는 상단에 유지)                                  │
└─────────────────────────────────────────────────────────┘
```

## 컴포넌트 구조

### 1. 페이지 (page.tsx) — Server Component

- `fetchAssignments()` 호출, 데이터를 통합하여 직렬화
- KPI 카드 렌더링 (배정 대학, 운영자, 개발자, 변경건)
- `<AssignmentExplorer>` 클라이언트 컴포넌트에 데이터 전달

### 2. AssignmentExplorer.tsx — Client Component (메인)

- 전체 분할 패널 레이아웃 관리
- 상태: `mode` ("operator" | "university"), `selectedId` (선택된 운영자/대학명), `search` (검색어)
- 왼쪽: `<LeftPanel>`, 오른쪽: `<RightPanel>`

### 3. LeftPanel.tsx — 왼쪽 패널

**구성:**
- 토글 버튼: `[운영자 | 대학]`
- 검색 입력 (현재 모드에 맞게 필터)
- 리스트

**운영자 모드 리스트:**
- 그룹별(1~6그룹) 구분 라벨
- 각 운영자: 이름 + 총 배정 건수
- 선택된 운영자 하이라이트 (primary 배경)
- 클릭 → `onSelect("operator", name)`

**대학 모드 리스트:**
- 가나다순 정렬
- 각 대학: 이름 + 대분류 뱃지 (4년제/전문대학 등)
- 변경된 대학은 빨간 도트 표시
- 클릭 → `onSelect("university", name)`

### 4. OperatorDetail.tsx — 운영자 상세 (오른쪽)

**프로필 헤더:**
- 운영자 이름 (font-black)
- 그룹 뱃지 + 총 배정 건수
- 유형별 건수 미니 타일 (수시 13 | 정시 13 | PIMS 5 | ...)

**유형별 섹션 카드:**
각 섹션은 접이식(기본 펼침):
- **수시** — 배정된 대학명 칩 나열 (대분류 색상 구분)
- **정시** — 동일 구조
- **재외국민/외국인/편입** — 해당 시 표시
- **대학원** — 대학원 배정 대학 칩
- **PIMS** — 전체사용/선택사용 구분 라벨
- **성적산출** — 대학 칩
- **상담앱** — 대학 칩

**대학 칩 클릭 → 대학 상세 뷰로 전환** (mode: "university", selectedId: 대학명)

### 5. UniversityDetail.tsx — 대학 상세 (오른쪽)

**대학 헤더:**
- 대학명 (font-black)
- 대분류 뱃지 + 지역 + 영업담당
- 변경 여부 표시

**배정 정보 섹션:**
현재 `UniversityRow` 구조와 유사하되 카드 대신 섹션 나열:
- 수시/정시: 운영자, 개발자 (각각 이름 표시)
- 대학원: 운영자, 개발자
- PIMS: 전체사용, 선택사용
- 성적산출: 운영자, 개발자
- 상담앱: 운영자, 개발자
- 비고 (remark)

**운영자명은 클릭 가능** → 클릭 시 해당 운영자 뷰로 전환 (mode: "operator", selectedId: 이름)

### 6. 양방향 탐색 플로우

```
운영자 리스트 → 운영자 상세 → 대학 칩 클릭 → 대학 상세 → 운영자명 클릭 → 운영자 상세
     ↑                                                              │
     └──────────────────────────────────────────────────────────────┘
```

왼쪽 패널은 항상 표시. 오른쪽만 전환. 왼쪽 토글/리스트 선택으로 언제든 다른 항목으로 이동 가능.

## 데이터 흐름

1. **page.tsx** (서버): `fetchAssignments()` → 대학별 통합 데이터 (`UniversityData[]`) + 운영자 요약 (`OperatorSummary[]`) 생성
2. **AssignmentExplorer** (클라이언트): props로 받아 상태 관리
3. 왼쪽/오른쪽 패널은 부모 상태(`mode`, `selectedId`)에 따라 렌더링
4. 서버 요청 없이 클라이언트에서 모든 탐색 처리 (데이터 이미 로드됨)

## 파일 구조

```
src/app/(dashboard)/operations/assignments/
├── page.tsx                  # Server: 데이터 fetch + KPI + Explorer 렌더
├── AssignmentExplorer.tsx    # Client: 분할 패널 레이아웃 + 상태 관리
├── LeftPanel.tsx             # Client: 토글 + 검색 + 리스트
├── OperatorDetail.tsx        # Client: 운영자 상세 (유형별 카드)
└── UniversityDetail.tsx      # Client: 대학 상세 (원카드 통합)
```

## 스타일 가이드

- 프로젝트 디자인 시스템 준수 (surface-container, primary, outline-variant 등)
- 왼쪽 패널: `w-[300px] shrink-0`, 오른쪽: `flex-1 overflow-y-auto`
- 토글 버튼: PIMS 탭 토글과 동일 스타일 (bg-primary text-on-primary 활성)
- 유형 뱃지: 기존 SECTION_TAGS 패턴 활용
- 대학 칩: 기존 WizardStep3 칩 스타일
- 선택된 리스트 항목: primary 배경 하이라이트
- 운영자명 클릭 가능 표시: hover 시 text-primary + underline

## 제거 항목

- `AssignmentFilters.tsx` (현재 검색만 있음) → LeftPanel에 통합
- `AssignmentViewTabs.tsx` → 사용 안 함
- 기존 OPERATOR ROSTER 테이블 → LeftPanel 운영자 리스트로 대체
- 기존 UNIVERSITY INDEX 그리드 → LeftPanel 대학 리스트로 대체
- 검색 결과 카드 뷰 → UniversityDetail로 대체
